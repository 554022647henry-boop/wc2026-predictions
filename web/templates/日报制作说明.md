# 日报制作说明

## 数据来源

- **今日结果**：`data/results/results.json`（按 `recorded_at` 过滤日期）
- **比赛预测**：`data/predictions/{match_id}/initial_prediction.json`
  - 用 `output.prediction` / `output.confidence` / `output.reasons` / `output.key_risk`
  - 胜平负概率用 `_internal.p_a_win` / `p_draw` / `p_b_win`
- **赛程时间**：`data/fixtures.json` → `kickoff_cst`（北京时间，按此升序排列）

---

## 结构（固定顺序）

1. **Hero**：日期 + 今日命中率 + 一句话总结 + 数据行（命中/明日场次/置信度分布）
2. **明日预测**（SECTION 01）：按 kickoff_cst 升序排列，置信度 badge + 概率条 + 信心对比条 + 依据 + 风险块
3. **今日复盘**（SECTION 02）：今日赛果（命中绿框/失败红框）+ Insight 引语
4. **失败复盘**（SECTION 03）：编号分析，结尾加琥珀色教训标签（仅当日有失败时显示）
5. **Footer**：`@Ligo哥 · AI预测系统`

### 顺序说明

预测先行的逻辑：用户打开日报最先想看的是"明天怎么押"，而不是回顾已经打完的比赛。赛果复盘放在预测之后作为参考背景，失败拆解放在最后供深度阅读。

### ⚠️ 完整性硬要求

- **今日结果**：必须包含当日所有已完赛场次，一场不漏
- **明日预测**：必须包含次日全部场次，一场不漏，按 `kickoff_cst` 升序排列
- 场次数量在 Hero 副标题中提及（如"4场比赛3中1失"）

### 全中情况处理

当天所有预测全部命中时，第4节**不写失败复盘**，改为：

```html
<!-- 替换 SECTION 02 -->
<div class="section-head r">
  <div class="section-label">SECTION 02</div>
  <div class="section-title r r-d1">今日亮点</div>
</div>
<div class="points">
  <div class="point r r-d1">
    <div class="point-num" style="color:var(--green)">★</div>
    <div class="point-content">
      <h3><!-- 最值得说的一场，分析为何命中 --></h3>
      <p><!-- 简短分析 --></p>
    </div>
  </div>
</div>
```

全错情况同样保留第4节，照常复盘，序号颜色保持红色。

## Hero 副标题写法

副标题（`.hero-sub`）每日手写，**1行以内**，一句话总结今日表现：

> `[今日最大亮点/意外]。[核心数据]`

示例：
- `"法国3–0、阿根廷2–0、阿尔及利亚2–1全中。挪威对塞内加尔低置信失手。"`
- `"强队命中率100%，平局识别依然是盲区。"`
- `"瑞士4-1大胜、加拿大6-0血洗，今日火力全开。"`

**数据行**（`.hero-data`）自动生成，格式固定：
```
命中 {X}/{N} · 明日 {M}场 · 高置信 {H}/{M} · 中置信 {C}/{M} · 低置信 {L}/{M}
```

> 数据行用 Space Mono 等宽字体，点分隔符居中。置信度计数仅统计明日预测场次。

## Insight 引语写法

Insight（`.insight blockquote`）是当日**最核心的一句话教训**，AI 提炼后人工审核。

规则：
- 必须是**一句话**，不加标点结尾
- 说的是**今日的系统性规律**，不是某场比赛的具体描述
- Syne 字体加粗，22px，`--blue-dark` 色
- `.insight-source` 是补充说明，13px 斜体，写触发这个结论的具体场次

示例：
```
blockquote: "强队纸面碾压的比赛，弱队的铁桶阵和小失误率比赔率更值钱"
source:     "两次失败的预测均涉及实力差距悬殊的对局"
```

## 视觉规范（与 STYLE_GUIDE.md 统一）

### 字号统一

| 元素 | 字号 |
|------|------|
| Insight blockquote | **22px**，Syne 700，`--blue-dark` |
| 正文理由 | 13px，Figtree，`--ink-2` |
| 各类标签/数据 | 10–12px，Space Mono |

### Scroll Reveal 统一

```js
const io = new IntersectionObserver(entries => {
  entries.forEach(e => {
    if (e.isIntersecting) { e.target.classList.add('on'); io.unobserve(e.target); }
  });
}, { threshold: 0.12 });  // ← 统一用 0.12，不用 0.1
```

### 失败复盘序号颜色

`.point-num` 在日报中**统一用红色** `var(--red)`（STYLE_GUIDE 里写的"蓝色"是视频模板用的，日报里红色更直观）。已在 STYLE_GUIDE 里注明此例外。

### 比分书写

比分用 **en dash**（`–`），不用连字符（`-`）：
```html
<div class="match-score">4–1</div>   ✅
<div class="match-score">4-1</div>   ❌
```

## 预测依据 Emoji 分类对照表

每条预测依据前需加 emoji 图标，让依据类型一目了然。根据 reason 文本中的关键词匹配选择对应 emoji：

| 类别 | Emoji | 触发关键词 |
|------|-------|-----------|
| 进攻/中场 | ⚽ | 调度、直塞、传中、射门、锋线、攻击、前插、破门、终结 |
| 防守 | 🛡️ | 防守、防线、后防、拦截、铁桶、高空球、对抗、盯防 |
| 数据/战术 | 📊 | 控球、xG、射门数、传球、战术、阵型、定位球、角球 |
| 历史/经验 | 📋 | 经验、历史、交锋、淘汰赛、大赛、心理素质、纪律 |
| 状态/气势 | 📈 | 状态、连胜、近况、火热、气势、势头、压迫 |
| 场地/主场 | 🏟️ | 主场、客场、高原、场地、天气、炎热、球迷 |
| 伤病/阵容 | 🏥 | 伤病、缺阵、受伤、复出、阵容、轮换 |
| 排名/实力 | 📊 | 排名、实力、身价、纸面、档次 |

> 若一条理由同时匹配多个类别，优先使用第一个匹配项。无匹配项时使用 `📌` 作为默认图标。
