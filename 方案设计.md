# 世界杯2026预测系统 — 完整方案设计

## 整体流程

```
系统启动
  ↓
全部104场比赛初始预测（用已有公开信息）
  ↓
[每场比赛独立触发以下流程]
  ↓
赛前24h → 搜集+预测
赛前12h → 搜集+预测
赛前2h  → 搜集+预测
赛前30min → 搜集+最终预测（封盘）
  ↓
更新HTML
  ↓
赛后录入实际结果 → HTML显示预测准确率
```

## 第一部分：信息搜集层

### Agent 1 — 信息搜集

**触发时机：**
- 赛前24h、12h、2h、30min各触发一次
- 首发阵容公布后立即额外触发一次
- 比赛前10分钟停止搜集（封盘）

**10个信息源：**
1. BBC Sport — bbc.com/sport/football
2. Goal.com — goal.com
3. ESPN FC — espn.com/soccer
4. Transfermarkt — transfermarkt.com（伤停、阵容价值）
5. Sofascore — sofascore.com（球员近期评分、状态）
6. UEFA.com — 官方新闻
7. FIFA.com — 官方阵容
8. WhoScored — whoscored.com（详细数据统计）
9. Sky Sports — skysports.com/football
10. The Guardian Football — theguardian.com/football

**每次搜集内容：**
- 两队最新伤停/停赛名单
- 首发阵容（如已公布）
- 近期状态新闻
- 赛前发布会/教练表态
- 赔率变动（Pinnacle/Bet365）
- 任何突发信息

---

### Agent 2 — 信息审阅 + 缺口补充

**输入：** Agent1搜集的原始信息

**流程：审阅 → 发现缺口 → 指派补充（最多2轮）**

```
Agent1搜集完
    ↓
Agent2第一轮审阅
  → 过滤无关内容
  → 检查关键信息是否齐全
  → 发现缺口，列出补充清单
    ↓
若有缺口 → 指派Agent1针对性补充（第1轮）
    ↓
Agent2第二轮审阅
  → 若仍有缺口 → 再指派Agent1补充（第2轮）
    ↓
无论第2轮结果如何，强制结束
用现有信息传给预测层
```

**Agent2检查的关键信息清单（必须覆盖）：**
- 两队核心球员伤停/停赛状态
- 首发阵容（若距比赛不足1小时）
- 近期状态（至少最近3场结果）
- 当前赔率
- 教练赛前表态（若有发布会）

**补充指令示例：**
```
缺口发现：
- B队门将伤情未确认，请搜索 "[队名] goalkeeper injury"
- 未找到本场赔率，请访问 Pinnacle 查询
- 缺少A队教练赛前发布会内容，请搜索 "[教练名] press conference"
```

**最终输出结构化JSON：**
```json
{
  "match_id": "WC2026_QF_01",
  "collection_round": "T-2h",
  "team_a": {
    "confirmed_absences": [],
    "doubtful": [],
    "confirmed_lineup": null,
    "key_player_form": "",
    "latest_news": ""
  },
  "team_b": {},
  "odds": {
    "pinnacle": {"a": 1.75, "draw": 3.60, "b": 4.50},
    "implied_prob": {"a": "56%", "draw": "26%", "b": "20%"}
  },
  "notable_info": "",
  "info_completeness": "完整 / 基本完整 / 部分缺失"
}
```

## 第二部分：预测引擎（5个Agent）

### 架构

```
Agent2结构化信息
        ↓
┌─────────────────────────────────┐
│  三个专家Agent（独立分析）          │
│                                 │
│  Agent A: 实力 + 关键个人          │
│  Agent B: 战术 + 当前状态          │
│  Agent C: 大赛基因 + 心理          │
└─────────────────────────────────┘
        ↓
   魔鬼代言人Agent
   （专找热门队败因）
        ↓
      裁判Agent
   （按证据强度判决）
        ↓
  胜 / 平 / 负 + 3条理由
```

---

### Agent A — 实力 + 关键个人

**分析重点：**
- 双方整体实力差距（ELO、FIFA积分、阵容市值）
- 有没有"一个人能改变这场比赛"的球员（姆巴佩、梅西级别）
- 这名关键球员当前状态如何，对方能否有效限制他
- 双方首发阵容深度对比

**输出：** 哪队在此维度占优 + 核心理由（2-3句）

---

### Agent B — 战术 + 当前状态

**分析重点：**
- 双方战术风格相生相克（高位压迫 vs 低位反击等）
- 阵型对位有无明显错位
- 近5场竞赛场表现趋势（进攻效率、防守稳定性）
- 疲劳度、关键位置伤停对整体的影响

**输出：** 哪队在此维度占优 + 核心理由（2-3句）

---

### Agent C — 大赛基因 + 心理

**分析重点：**
- 这支队伍在大赛淘汰赛的历史基因（克罗地亚式的淘汰赛DNA）
- 绝境中的表现历史（落后能否翻盘）
- 这场比赛谁的心理压力更大、谁更需要赢
- 教练大赛指挥经验

**输出：** 哪队在此维度占优 + 核心理由（2-3句）

---

### 魔鬼代言人Agent

**任务：** 只找热门队失败的理由

**强制要求：** 必须给出至少2条具体的、有根据的热门队可能输的理由，不能泛泛而谈

**输出：** 2-3条热门队的败因风险

---

### 裁判Agent — 最终判决

**决策逻辑：按证据强度判（不数票）**

```
第一级（最重）：硬事实
 → 确认伤停/停赛、确认首发、近期客观数据
 → 可以推翻所有软因素

第二级：战术具体分析
 → 这场比赛里谁的打法克制谁

第三级（最轻）：软因素
 → 大赛历史、心理压力、传统优势
 → 不能推翻硬事实
```

**市场赔率使用方式：**
- 作为背景信息（"市场认为A队56%概率赢"）
- 不是锚点，有充分硬事实时可推翻市场判断
- 若判断与市场差距 > 20%，必须明确说明原因

**预测文件分两层存储：**

```json
{

  "round": "T-30min",
  "timestamp": "2026-07-15T20:30:00Z",

  "_internal": {

    "info_completeness": "基本完整（缺梅西伤情官方确认）",
    "agent_a_strength": {
      "favor": "法国",
```

**3条理由写作原则（面向普通球迷）：**
- ✅ 具体、直观、有画面感，提到具体球员或战术现象
- ❌ 不出现 xG、ELO、泊松分布等技术词汇

## 第三部分：HTML展示

### 页面内容

**主视图：** 世界杯完整赛程图（Bracket）
- 小组赛：12组 × 4队，每场显示预测
- 淘汰赛：完整对阵树，32强 → 冠军

**每场比赛卡片：**
```
[阿根廷] vs [法国]
2026-07-15  洛杉矶  决赛

预测：阿根廷胜  置信度：中
• 梅西确认首发，大赛经验无与伦比
• 阿根廷近5场仅失1球，防守极为稳固
• 整体体系成熟度高于法国

赛后 → 实际结果：阿根廷 1-0 法国  ✓ 预测正确
```

**顶部统计：** 总预测数 / 正确数 / 准确率

**预测历史：** 点击比赛可展开查看4轮预测的演变过程

### 部署
- 静态HTML，本地直接打开
- 或 Python 简单HTTP服务器，可局域网/公网访问

## 开发顺序

1. `data/fixtures.json` — 录入全部赛程
2. `agents/collector.py` + `agents/reviewer.py` — 搜集和过滤
3. `agents/predictor.py` — 5个预测Agent
4. `web/index.html` — 赛程图网页
5. `scheduler.py` — 定时触发
6. 回测验证 — 用2022年世界杯历史数据测试准确率

---

## 准确率预期

| 场景 | 预期准确率 |
|------|-----------|
| 强队对阵弱队 | ~70% |
| 势均力敌 | ~50% |
| 整体平均 | ~55-60% |

系统核心优势：比市场更快处理突发信息（首发/伤停），以及战术和心理层面的深度分析。
